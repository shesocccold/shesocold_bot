# Локальный Telegram-бот Ани

Минимальный локальный бот без хостинга, базы данных, Railway, Supabase, Docker и деплоя. Бот работает через long polling: пока открыт терминал и запущен `python bot.py`, бот живой.

## Что умеет

- разделяет `🍳 готовку` и `📍 места`
- сохраняет любимые рецепты и места локально в `favorites.json`
- открывает папины омлеты только после пароля
- ищет рецепты по словам, БЖУ, времени, фото и ингредиентам
- ищет места по городу, метро, типу, статусу и ключевым словам
- показывает фото рецепта или места, если оно добавлено

## Как запустить

1. Создай бота через [@BotFather](https://t.me/BotFather) в Telegram.
2. Скопируй токен.
3. В папке проекта создай файл `.env`.
4. Добавь в `.env` строку:

```env
BOT_TOKEN=токен_от_BotFather
```

5. Создай виртуальное окружение:

```bash
python -m venv .venv
```

6. Активируй его.

Windows:

```bash
.venv\Scripts\activate
```

macOS / Linux:

```bash
source .venv/bin/activate
```

7. Установи зависимости:

```bash
pip install -r requirements.txt
```

8. Запусти бота:

```bash
python bot.py
```

Если локальное окружение уже создано, на Windows можно запустить без активации:

```bash
run_bot.bat
```

Если всё хорошо, в терминале появится:

```text
бот запущен
```

После этого открой своего бота в Telegram и напиши `/start`.

## Главное меню

- `🍳 рецепты` — поиск, рандом, любимые рецепты, папины омлеты.
- `📍 места` — где была, куда хочу, любимые места, поиск заведений.
- `⭐ любимое` — быстрый вход в любимые рецепты и места.

Папины омлеты не показываются в общем каталоге, поиске и рандоме.

## Запросы по готовке

Можно писать боту обычными словами:

```text
ужин с курицей до 600 ккал
быстрое с сыром
без бекона
белка больше 20
жиров до 20
углеводов меньше 50
пп с фото
рандомный ужин без бекона
```

Бот понимает категории, ингредиенты, теги, слова из шагов рецепта, время, фото и примерные БЖУ.

## Запросы по местам

Примеры:

```text
места
кофе у метро Китай-город
бар где я была
хочу сходить на бранч
любимые места
куда сходить на пиццу
ресторан Таганская
```

Бот понимает город, метро, тип заведения, статус `была` / `хочу сходить`, любимые места и слова из названия, адреса, тегов и отзыва.
На широкий запрос вроде `места` или `все места` бот показывает быстрые фильтры. На конкретный запрос показывает короткий список кнопок, а полную карточку открывает только после выбора места.

## Если `python --version` показывает только `Python`

На Windows такое обычно значит, что вместо настоящего Python сработал алиас Microsoft Store.

Что сделать:

1. Установи Python 3 с сайта [python.org](https://www.python.org/downloads/).
2. Во время установки поставь галочку `Add python.exe to PATH`.
3. Закрой и заново открой терминал.
4. Проверь:

```bash
python --version
```

Нормальный результат выглядит примерно так:

```text
Python 3.12.5
```

## Как добавлять рецепты

Рецепты лежат в `recipes.json`. Это обычный массив объектов:

```json
{
  "title": "название",
  "category": "категория",
  "tags": ["тег1", "тег2"],
  "ingredients": ["ингредиент 1", "ингредиент 2"],
  "steps": ["шаг 1", "шаг 2"],
  "image": "assets/recipes/photo.jpg",
  "nutrition": {
    "calories": 350,
    "protein": 18,
    "fat": 22,
    "carbs": 25
  },
  "time_minutes": 15,
  "is_for_dad": true
}
```

Фото рецептов лучше хранить в папке `assets/recipes`.
БЖУ указываются примерно на одну порцию.

## Как добавлять места

Места лежат в `places.json`:

```json
{
  "id": "place-id",
  "title": "Название",
  "city": "Москва",
  "address": "улица, дом",
  "metro": ["Китай-город"],
  "types": ["coffee", "cafe"],
  "status": "wishlist",
  "favorite": false,
  "tags": ["кофе", "бранч"],
  "review": "мой отзыв",
  "photos": ["assets/places/photo.jpg"],
  "url": "https://yandex.ru/maps/..."
}
```

Статусы:

- `visited` — где я была
- `wishlist` — куда хочу сходить

Типы мест сейчас: `coffee`, `cafe`, `restaurant`, `bar`, `brunch`, `kebab`, `pizzeria`, `asian`, `culture`, `record_store`.
Фото мест лучше хранить в папке `assets/places`.

Настоящий токен не храни в коде и не отправляй в git. Файл `.env` уже добавлен в `.gitignore`.
